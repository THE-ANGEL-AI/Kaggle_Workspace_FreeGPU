Подробности по проекту

ComfyUI как пайплайн из шести модулей

Не одна библиотека — а рабочий конвейер: окружение, ноды, симлинки на модели, распределение по двум GPU и публичный туннель. Каждый модуль идемпотентен.

🚀

Запуск одной строкой

Три Python-скрипта в instal/: окружение, ноды, запуск. Через пару минут после старта получаете публичный URL на работающий ComfyUI.

~2 мин до публичного URL
🛡️

Самовосстановление venv

Kaggle при рестарте сессии ломает venv (битый симлинк, слетевший +x). Скрипты ловят это и чинят автоматически за секунды — без переустановки torch.

≈ 99% идемпотентность
⚡

torch cu130 под T4

Драйвер 580.x, нативный SDPA вместо нерабочего на Turing xformers.

🧩

DisTorch2 на 2× GPU

ComfyUI-MultiGPU распределяет слои между двумя T4 и CPU аккуратно.

📦

Модели через симлинки

Flux2 GGUF и LTX 2.3 из /kaggle/input. Один раз скопировал датасет — дальше мгновенно.

🔗

Публичный URL прямо из ячейки

Cloudflare-туннель (trycloudflare.com) поднимается из блокнота без белого IP и без проброса портов. Под ячейкой появляются кнопки: открыть ComfyUI, остановить процесс, перезапустить с новым URL — ядро Kaggle при этом не перезапускается. Keep-alive не даёт Kaggle усыпить сессию через 40 минут бездействия.

🌐

Flux2 GGUF, LTX 2.3 Director в workflows/ — готовые графы.

Быстрый старт

Три ячейки — и ComfyUI работает

Включите в Kaggle-блокноте GPU T4 ×2 и интернет. Потом вставьте ячейки по порядку.

kaggle-notebook.ipynb — ComfyUI setup 
# 0. Скрипты из репозитория (или обновить, если уже склонировано)
!git clone https://github.com/THE-ANGEL-AI/Kaggle_Workspace_FreeGPU.git || \
  git -C Kaggle_Workspace_FreeGPU pull

# 1. Окружение: uv + venv (Python 3.12) + torch cu130 + ComfyUI + Manager
!python Kaggle_Workspace_FreeGPU/instal/instal_comfyui.py

# 2. Кастомные ноды + симлинки на модели из /kaggle/input
!python Kaggle_Workspace_FreeGPU/instal/instal_castom_node.py

# 3. Запуск + Cloudflare-туннель + keep-alive кнопки
%run Kaggle_Workspace_FreeGPU/instal/start.py

Можно и одной строкой: %run .../instal/start.py сам проверит окружение и вызовет нужный установщик автоматически.
